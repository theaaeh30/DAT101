[What is the difference between invoking and creating a function?]

    [Å opprette en funksjon handler om å definere hva funksjonen skal gjøre, uten at den faktisk kjører.

    Å kalle funksjonen vil derimot utføre de handlingene som er beskrevet i funksjonen. Det er som å skrive en oppskrift (opprette) og å lage maten (kalle).]

    [Explain the syntax regarding semicolons and code blocks.]

    [Semikolon brukes som regel til å avslutte en setning, mens kodeblokker, som er omsluttet av {}, samler flere setninger.

    Inne i kodeblokker er semikolon ofte frivillige, men utenfor blokker er de som oftest påkrevd. ]

    [What happens when we try to store values from a function when it doesn't return any?]

    [Hvis en funksjon ikke returnerer noe, vil en variabel som prøver å lagre funksjonens resultat få verdien undefined.

    Dette skyldes at funksjoner automatisk returnerer undefined når ingen returverdi er spesifisert. ]

    [How do we separate parameters from each other when creating a function?]

    [Når man definerer en funksjon, skilles parametere med komma inne i parentesene.

    Eksempelvis i function eksempel(param1, param2), er param1 og param2 adskilte parametere. ]

    [Explain how parameter names are related during the invocation and creation of a function]

    [Ved opprettelse fungerer parameternavn som plassholdere for verdier som sendes inn senere.

    Når funksjonen kalles, sendes verdier (argumenter) i samme rekkefølge, slik at de samsvarer med parametrene.]

    [Explain why a parameter is undefined when a function call is missing a parameter.]

    [Hvis man kaller en funksjon uten å sende inn alle argumenter, blir manglende parametere automatisk undefined. Dette er fordi det ikke gis noen verdi til disse parameterne. ]

    [Explain what happens when we use recursive functions.]

    [Rekursive funksjoner kaller seg selv med endrede argumenter til de når en basistilstand.

    Uten en riktig basistilstand kan de skape en uendelig løkke og føre til stack overflow. ]
